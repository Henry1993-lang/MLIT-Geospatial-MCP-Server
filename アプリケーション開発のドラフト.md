# MCPを利用した不動産鑑定士向けアプリケーション開発ドラフト

## 1. コンセプト

本アプリケーションは、不動産鑑定士が対象不動産の調査、比較事例の抽出、価格形成要因の確認を行う際に、MLIT Geospatial MCP Serverを通じて公的な地理空間情報を自然言語で取得・活用できる業務支援ツールである。

単にレポートを自動生成するのではなく、鑑定士が地図やチャットを通じて対象地を指定し、AIエージェントがMCPを介して必要なデータを取得し、調査・判断・比較の作業を支援することを目的とする。

## 2. 想定ユーザー

- 不動産鑑定士
- 鑑定評価補助者
- 不動産調査会社
- 金融機関の担保評価担当者
- 不動産デューデリジェンス担当者

## 3. 中核アイデア

### 地図連動AIワークスペース

鑑定士が地図上で対象地を選択し、チャットで調査内容を指示すると、AIエージェントがMLIT Geospatial MCP Serverを利用して必要な地理空間情報を取得する。

例:

- 「この土地の周辺取引事例を過去3年で探して」
- 「用途地域と防火地域を確認して」
- 「洪水、津波、土砂災害のリスクを確認して」
- 「駅距離と駅別乗降客数を踏まえて商業地としての立地を見て」
- 「この事例と対象地の違いを整理して」

アプリケーションは、MCPを単なるデータ取得APIとしてではなく、AIエージェントが鑑定士の調査意図を理解して必要なデータを取りに行くための実行基盤として利用する。

## 4. 主な機能

### 4.1 対象地の指定

- 住所検索
- 緯度経度入力
- 地図上クリック
- 地番・住居表示からの候補選択
- 複数地点の比較

### 4.2 MCP経由のデータ取得

MLIT Geospatial MCP Serverを通じて、以下のような情報を取得する。

- 不動産価格情報
- 鑑定評価書情報
- 地価公示・地価調査ポイント
- 用途地域
- 都市計画区域・区域区分
- 立地適正化計画
- 防火・準防火地域
- 地区計画
- 高度利用地区
- 駅別乗降客数
- 小学校区・中学校区
- 学校、保育園、医療機関、福祉施設
- 将来推計人口250mメッシュ
- 人口集中地区
- 洪水浸水想定区域
- 高潮浸水想定区域
- 津波浸水想定
- 土砂災害警戒区域
- 災害危険区域
- 液状化発生傾向
- 大規模盛土造成地
- 地すべり防止地区
- 急傾斜地崩壊危険区域

### 4.3 自然言語による調査指示

鑑定士はAPI名やGIS操作を意識せず、自然言語で調査を依頼する。

例:

- 「対象地から1km以内の住宅地取引を抽出して」
- 「近隣地域の標準地と基準地を比較して」
- 「災害リスクが価格に影響しそうか確認して」
- 「類似性の高い取引事例を5件に絞って」
- 「駅距離、用途地域、地積が近い順に並べて」

AIエージェントは指示を解釈し、必要なMCPツールを選択して実行する。

### 4.4 比較事例の探索と絞り込み

対象地と周辺の取引事例・公示地・基準地を比較し、鑑定士が採用候補を検討しやすい形で表示する。

比較軸:

- 対象地からの距離
- 取引時点
- 用途地域
- 土地利用
- 地積
- 形状
- 前面道路
- 最寄駅・駅距離
- 価格水準
- 災害リスク
- 周辺施設
- 人口動向

AIは候補を自動で採否判断するのではなく、「比較可能性が高い理由」「注意すべき差異」「追加確認が必要な点」を提示する。

### 4.5 地図・チャット・テーブルの連動

地図上に以下を重ねて表示する。

- 対象地
- 取引事例
- 地価公示・地価調査地点
- 用途地域
- 災害リスク区域
- 駅・公共施設
- 人口メッシュ

チャットで条件を変更すると、地図表示と比較事例・取得データテーブルが同時に更新される。テーブルは単なる結果一覧ではなく、地図とチャットに連動する操作面として扱う。

例:

- 「洪水浸水想定区域内の事例を除外して」
- 「駅から500m以内だけ表示して」
- 「商業地域と近隣商業地域の事例だけ残して」

連動仕様:

- 地図上の取引事例・公示地・基準地をクリックすると、該当行がテーブルで選択される
- テーブル行をクリックすると、地図が該当地点へ移動し、チャットにその事例の要約と確認候補が表示される
- チャットで絞り込み条件を追加すると、地図の表示地点とテーブル行が同じ条件で更新される
- テーブルのフィルタやソートを変更すると、地図上の表示・強調も同時に切り替わる
- 地図上で範囲選択すると、その範囲内の事例だけがテーブルに表示される
- チャットで「この3件を比較して」と指示すると、テーブルで選択中の行を対象にAIが比較する
- テーブルの各行には、MCPで取得した元データ、出典、取得時点、該当する地図レイヤーを紐づける
- AIの要約・推論から、根拠となるテーブル行と地図上の地点へ戻れるようにする

比較事例・取得データテーブルの役割:

- 取得した公的データの一覧
- 地図上の地点・区域レイヤーとの対応確認
- 比較事例の採用候補管理
- 除外理由や鑑定士メモの記録
- AIが参照した根拠データの確認
- チャット指示の対象行選択

### 4.6 価格形成要因の確認支援

MCPで取得したデータをもとに、価格形成要因を整理する。

分類例:

- 交通接近条件
- 行政的条件
- 環境条件
- 災害・地盤リスク
- 公共施設接近性
- 商業集積性
- 人口動向
- 需給動向

鑑定士が確認すべき論点を提示し、必要に応じて追加調査を促す。

- 社内鑑定システムへのAPI連携

## 5. 画面構成案

### 5.1 メイン画面

- 左側: 地図
- 右側: AIチャット
- 下部: 地図・チャットと連動する取得データ・比較事例テーブル
- 上部: 対象地検索、調査目的、保存、出力

メイン画面では、地図、AIチャット、取得データ・比較事例テーブルを同一の調査セッションとして扱う。いずれかの領域で対象地点、条件、選択行が変わると、他の領域も同じ状態に同期する。

### 5.2 対象地パネル

- 所在地
- 緯度経度
- 用途地域
- 都市計画情報
- 最寄駅
- 災害リスク概要
- 周辺価格指標

### 5.3 比較事例パネル

- 取引事例一覧
- 公示地・基準地一覧
- 類似度スコア
- 採用候補フラグ
- メモ
- 除外理由
- 地図上の表示状態
- チャットで参照中の行
- MCP取得元データへのリンク
- 出典・取得時点

比較事例パネルの操作:

- 行選択: 地図上の地点をハイライトし、チャットの文脈に追加する
- 複数行選択: AIに比較・差異整理・採否理由案の作成を依頼できる
- フィルタ: 地図レイヤーとチャットの検索条件に反映する
- ソート: 類似度、距離、時点、価格水準などで並び替える
- 採用候補フラグ: チャットでの追加分析対象として固定する
- 除外理由: 地図上では薄く表示し、チャットでは除外済みとして扱う

### 5.4 取得データパネル

- MCPツール名
- データ種別
- 対象地点・対象範囲
- 取得結果
- 出典
- 取得時点
- 地図レイヤー
- チャットでの参照有無
- エラー・欠落情報

取得データパネルでは、AIがどのMCPデータを根拠に回答したかを確認できる。チャット内の回答から取得データ行へ遷移でき、取得データ行から地図レイヤーの表示・非表示を切り替えられる。

### 5.5 リスク・規制パネル

- 災害区域の該当状況
- 都市計画規制
- 防火・準防火地域
- 地区計画
- 高度利用地区
- 追加確認事項

## 6. MCP利用の設計方針

### 6.1 AIエージェントの役割

AIエージェントは、鑑定士の自然言語指示を以下に分解する。

1. 対象地の特定
2. 必要なデータカテゴリの判断
3. MCPツールの選択
4. データ取得
5. 取得結果の要約
6. 地図・テーブル・チャット文脈への反映
7. ユーザー操作による選択状態の同期
8. 次に確認すべき論点の提示

### 6.2 MCP呼び出しの考え方

- ユーザーの質問に応じて必要なAPIだけを呼び出す
- 取得データの出典と取得時点を保持する
- 同一地点・同一条件の結果はキャッシュする
- 鑑定士が後から根拠を確認できるように、MCPの呼び出し履歴を保存する
- AIの推論と公的データの事実を画面上で明確に区別する
- MCP取得結果には、地図上の地点・区域、テーブル行、チャット回答を結びつける内部IDを付与する
- チャット回答には、参照したテーブル行と地図レイヤーへ戻れる根拠リンクを付ける

### 6.3 ガードレール

- AIの出力は鑑定評価そのものではなく、調査支援・論点整理として扱う
- 最終判断は不動産鑑定士が行う
- 取得データの欠落、古さ、APIエラーを明示する
- 不確実な推論は断定しない
- 利用規約、プライバシーポリシー、免責事項を遵守する

## 7. MVP案

### MVPの目的

最初のMVPでは、鑑定士が対象地を指定し、周辺の価格情報・都市計画・災害リスクを自然言語で取得できることを検証する。

### MVP機能

- 住所または地図クリックによる対象地指定
- チャットによる調査指示
- MCP経由で以下を取得
  - 地価公示・地価調査
  - 不動産価格情報
  - 用途地域
  - 駅別乗降客数
  - 洪水・土砂災害等の災害リスク
- 地図上への結果表示
- 比較事例テーブル表示
- 取得データテーブル表示
- 地図・チャット・テーブルの選択状態同期
- AIによる論点整理
- 取得データの出典表示

### MVPで検証すること

- 鑑定士が自然言語で必要なデータを取得できるか
- 比較事例探索の時間を短縮できるか
- MCP経由のデータ取得が業務フローに耐えるか
- 地図・チャット・テーブルの連動が実務上使いやすいか
- AIの説明が鑑定士の判断を妨げず、補助として機能するか
